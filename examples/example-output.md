# Example Output

## User Prompt
“为什么这个创业团队明明人很多、开会很多，却总像在原地打转？”

## Structural Judgment
这个团队的核心问题不像是“没有人努力”，更像是扩张速度已经超过了当前结构的承载能力。表面上成员和会议都在增加，但真正承担整合、决策与跨层翻译的角色没有同步长出来，所以系统变得更嘈杂，却没有更有效。

## Core Contradiction
团队想要的是规模化推进，但依然在使用适合小团队的协调方式。增长发生了，结构没有跟上，于是沟通越多，失真和耗散反而越大。

## Main Risk Direction
如果继续靠加人、加会和加任务推进，而不重建角色分层与决策路径，团队最可能进入的是“忙碌型停滞”：看上去很热闹，实际上逐步空心化，最后核心成员疲惫、外围成员失去方向感。

## Best Intervention Priority
优先修复“谁负责整合与决策翻译”这一层，而不是继续扩充任务列表。先建立最小清晰分工、决策界面和信息回路，才能让现有投入真正产生推进力。

## Uncertainty / Alternative Readings
如果外部市场或资金窗口正在快速变化，那么一部分混乱也可能来自环境压力而非纯内部结构问题。还需要确认：团队是否其实存在隐藏的双锚点冲突，例如有人追求增长，有人追求产品纯度。
