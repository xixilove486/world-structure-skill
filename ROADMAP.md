# 路线图

这份路线图的目标，是让 `world-structure-skill` 同时满足两件事：

- 作为作品阅读时，结构清楚、表达完整
- 作为 Codex skill 安装时，路径稳定、维护方便

## 第一阶段：稳定结构

当前重点：

- 保持可安装内容集中在 `skills/world-structure/`
- 保持 `skills/world-structure/SKILL.md` 为主入口
- 保持根目录文档承担展示和仓库说明功能

## 第二阶段：增强使用体验

后续可以继续做：

- 打磨模板
- 增补更强的示例输出
- 增加适合第一次使用者的快速上手说明
- 继续统一 skill 包内部路径与术语

## 第三阶段：增强公开展示

可以考虑：

- 继续优化 GitHub 首页呈现
- 增加简短中文导语与简短英文导语
- 增加一张安装流程或目录结构示意图
- 增加常见问题说明

## 第四阶段：扩展版本

未来如有需要，可以继续探索：

- 领域化变体
- 更轻量版或更完整版
- 多语言版本
- 配套的衍生 skill
