# 核心概念

本文件定义世界模型使用的主要结构概念。

这些概念不是用来攻击人的标签。
它们是用来分析系统如何维持、漂移、硬化、碎裂、更新或崩解的工具。

## 1. 系统（system）

系统，是一个有边界但从来不可能完全密封的关系、角色、记忆与压力模式。

系统可以是：
- 一个团队
- 一段关系
- 一个项目
- 一个家庭
- 一个社群
- 一个组织
- 一个具有重复结构的个人处境

第一步永远是：界定真正正在被分析的系统是什么。

系统不只是部件或人的集合。
它更是以下要素共同形成的模式：
- 部件如何相互关联
- 什么让它们保持协调
- 什么压力在塑形
- 它们承接了什么记忆
- 它们试图维持什么，或者变成什么

## 2. 锚点（anchor）

锚点，是系统显性或隐性围绕其运转的中心。

它可能是：
- 一个目的
- 一个承诺
- 一个使命
- 一个共同保护目标
- 一种身份认同
- 一种现实必要性
- 一个隐藏的组织中心

锚点可能处于以下状态：
- 清晰
- 分裂
- 空心化
- 被替换
- 仅作装饰

系统可以带着张力存活。
但很难在锚点彻底丧失后长期维持。

锚点之所以重要，是因为它决定：
- 系统认为什么值得保护
- 系统接受什么代价与权衡
- 哪种漂移被视为危险
- 哪种修复仍然可能

## 3. 约束（constraint）

约束，是系统在承载、协调、更新或转化上遇到的真实上限。

例如：
- 时间
- 金钱
- 信任
- 记忆
- 合法性
- 能量
- 对复杂度的承受力
- 环境压力
- 带宽
- 替代能力

约束不是借口。
它是系统真正能承受什么的边界。

糟糕的诊断，常会把结构性约束误读成懒惰、不忠诚或意志不足。
更好的诊断会问：
- 这个系统实际背了多重的负荷
- 它失去了哪些承载能力
- 哪些问题不是靠施压就能解决的

## 4. 层级（layer）

层级，是系统内部一个有意义的分析层面。

常见层级包括：
- 核心 / 中层 / 边缘
- 决策 / 执行 / 环境
- 可见 / 不可见
- 正式 / 非正式
- 维持 / 变化

系统可能因为某个层级塌了而失衡，也可能因为层级之间的翻译断了而失衡。

层级分析重要，是因为很多问题并不发生在单一层级内部，而是发生在层级之间的连接失败上。

例如：
- 决策无法转译成执行
- 核心认同不再触达边缘
- 非正式实践抵消了正式设计
- 表面系统掩盖了另一套实际运作的隐性系统

## 5. 角色（role）

角色，是功能位置，不是性格标签。

重要角色类型包括：
- 维持性角色
- 变化推动角色
- 连接角色
- 缓冲角色
- 记忆承载角色
- 阻塞角色
- 抽取型角色

同一个人可能承担多个角色。
糟糕的结构，常常让一个人背上太多彼此冲突的角色。

角色分析有用，是因为很多问题未必来自“坏人”，而可能来自：
- 必要角色缺位
- 不健康角色被奖励
- 某个人承担了过多互相冲突的功能
- 系统在惩罚它真正依赖的角色

## 6. 推力链（thrust chain）

推力链，是维持系统存活、运转与回流的那串关键补给关系。

推力可能来自：
- 资源
- 信任
- 合法性
- 情感投入
- 制度记忆
- 声誉
- 仪式
- 更新路径
- 替代路径
- 社会或物质基础设施

很多系统在一条隐藏推力链断掉之前，看起来都还很稳定。

推力链往往部分不可见。
一个系统之所以暂时看起来很强，可能是因为：
- 某个人在默默吸收巨大的协调成本
- 旧有信任仍在替新弱点兜底
- 制度记忆还在被过去借用
- 情感承诺还在掩盖结构脆弱

当推力链开始变薄时，系统通常会先变得更吵、更乱，然后才显性崩解。

## 7. 扰动（disturbance）

扰动，是任何会持续削弱锚点、扭曲传导、侵蚀信任，或抬高协调成本却没有带来更新收益的因素。

扰动可能来自：
- 某个人
- 某种角色配置
- 某种激励
- 某个内部矛盾
- 外部压力
- 一条僵化规范
- 阶段错配

并非所有分歧都是扰动。
正确的问题不是：
- 谁在反对我？

而是：
- 什么在持续削弱系统保持连贯的能力？

扰动看起来之所以强，有时并不是它本身绝对强，而是因为：
- 锚点已经偏弱
- 推力链已经变薄
- 角色已经混乱
- 层级翻译已经断裂
- 系统已经进入高脆弱阶段

## 8. 漂移（drift）

漂移，是系统缓慢偏离其功能中心的过程。

它常表现为：
- 锚点空心化
- 仪式替代功能
- 抽取替代更新
- 控制替代信任
- 增长替代连贯
- 生存替代目的

漂移危险，是因为它不一定伴随显性崩溃。

系统表面上仍然活着，但实际上可能已经发生：
- 意义变薄
- 协调变成表演
- 形式仍在，原有功能却淡了
- 人们还在参与，却已经不是为了同一个理由

漂移中的系统，往往最难诊断，因为它从外面看仍然像它自己。

## 9. 熵增（entropy）

熵增，是系统如果没有持续更新，就会越来越昂贵、嘈杂、僵硬、碎裂或空转的倾向。

它可能表现为：
- 官僚性增厚
- 反复误解
- 记忆流失
- 角色混乱
- 集体疲惫
- 象征维持替代真实功能
- 越来越费力，却越来越不连贯

熵增是常态。
真正的问题是：系统靠什么更新。

一个没有更新路径的系统，可以靠惯性、恐惧、习惯或抽取短期存活，但会越来越脆。

## 10. 阶段（stage）

阶段，是系统大致所处的结构发展位置。

常见阶段包括：
- 形成期
- 扩展期
- 瓶颈期
- 过渡期
- 稳定期
- 僵化期
- 碎裂期
- 更新尝试期

阶段不是命运。
它是一种帮助我们更准确解释症状的近似位置判断。

同样的症状，在不同阶段含义不同。

例如：
- 形成期的失序，可能代表开放性
- 碎裂期的失序，可能代表系统正在散开
- 稳定期的刚性，可能是在保护连贯性
- 衰退期的刚性，可能是在阻断更新

因此，阶段判断不是装饰项。
它会改变我们对现象的理解方式。

## 11. 核心矛盾（core contradiction）

核心矛盾，是那个持续生成多个表面症状的深层张力。

例如：
- 增长速度跑赢了结构承载
- 控制欲跑赢了信任
- 抽取跑赢了更新
- 身份叙事跑赢了现实
- 速度跑赢了记忆
- 形式还活着，意义却已经变薄

一个强的诊断，通常会尽早抓住核心矛盾。

如果抓不住，分析就容易变成症状清单，没有中心。

核心矛盾之所以有用，是因为它能回答：
- 表面问题的共同根在哪里
- 为什么同类失衡会反复出现
- 干预应该落在哪个杠杆点上

## 12. 干预优先级（intervention priority）

干预优先级，是那个最该先修、且结构杠杆率最高的修复点。

它不一定是：
- 最吵的问题
- 最戏剧化的问题
- 最能让人情绪满足的问题

系统最快改善，往往发生在第一修复点命中了：
- 真正瓶颈
- 关键的信任断点
- 破掉的层级翻译
- 更新缺口
- 空心化的锚点

干预优先级是这个模型最重要的实际用途之一。

重点不是先解释一切。
重点是找出：修了哪里，其他部分会跟着松动。

## 最后说明

这些概念是用来澄清结构的，不是用来冻结现实的。

它们应当让复杂处境更可读，而不是让分析者更自以为是。
