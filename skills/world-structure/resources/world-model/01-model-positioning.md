# 模型定位

世界模型不是世界本身。

它是一套结构分析框架，用来观察复杂系统如何形成、维持、漂移、硬化、碎裂、更新或崩解。

它适合分析：

- 团队
- 关系
- 项目
- 社群
- 组织
- 具有重复模式的个人处境
- 出现隐藏结构、反复冲突、停滞、漂移或失衡的复杂系统

## 这个模型用来做什么

世界模型主要适合四类任务：

1. 诊断  
   判断这是什么类型的系统，它当前处于什么状态。

2. 归因  
   理解当前结构模式为什么会形成。

3. 风险方向判断  
   估计如果不调整，系统最可能朝哪个方向漂移、失衡或断裂。

4. 干预优先级判断  
   找出最该先修什么，以及为什么要先修那里。

## 这个模型不用于什么

这个模型不适合用来：

- 预测精确未来事件
- 取代法律、医疗或财务专业判断
- 做临床诊断
- 在证据薄弱时宣布某个人的“真实本质”
- 用结构语言去攻击或压制某个人
- 假装所有问题都能靠分析解决

## 核心姿态

这个模型把复杂处境看成**在运动中的结构**。

它关心的是：

- 什么正在被维持
- 什么正在漂移
- 系统依赖什么
- 系统承受不了什么
- 什么在为它补给
- 什么在让它变形
- 它大致处于哪个阶段
- 哪类失衡或更新正在变得更可能

因此，模型优先观察：

- 锚点（anchor）
- 约束
- 层级
- 角色
- 推力链（thrust chain）
- 扰动
- 漂移
- 熵增
- 阶段
- 干预优先级

## 工作态度

这个模型应当被有纪律地使用。

它应当保持：

- 结构优先，而不是先发责备
- 清楚克制，而不是戏剧化
- 实用有限，而不是假装绝对
- 对不确定性诚实
- 对替代解释保持警觉

## 最后说明

世界模型是一面镜子，不是一把锤子。

它的目的不是赢下争论，也不是给人贴标签。
它的目的，是帮助一个系统对自己变得更可见。
