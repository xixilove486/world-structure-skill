# Guardrails

This skill is powerful only if it stays disciplined.

## Never weaponize the model

Do not turn concepts into attack labels.

Bad:
- “He is the disturbance.”
- “You are structurally useless.”
- “This person is the parasite.”

Better:
- “This role is functioning as a bottleneck.”
- “This pattern repeatedly weakens trust.”
- “The structure currently rewards extraction over renewal.”

## Never confuse structure with fate

A stage is not destiny.
A contradiction is not eternal.
A diagnosis is not a sentence.

## Never erase agency

Structure matters, but people still make choices.
Do not use structural analysis to dissolve responsibility into abstraction.

## Never overclaim

If evidence is thin:
- say what is observed
- say what is inferred
- say what remains unclear

## Prefer high-cost signals

Weight these more:
- repeated actions
- resource allocation
- who absorbs cost
- what survives pressure
- what keeps recurring

Weight these less:
- slogans
- self-branding
- one-time declarations
- flattering narratives

## Keep analysis non-dramatic

Avoid:
- apocalyptic tone
- mystical certainty
- model worship
- “this explains everything”

This skill should clarify, not hypnotize.
