# 世界结构.skill

> 把复杂对象当作**结构系统**来分析，而不是当作一团偶然事件、情绪冲突或单个人的问题。

`世界结构.skill` 是一个面向 AI 助手的结构分析技能包。  
它适合处理团队、关系、项目、制度、社群、叙事系统与复杂处境，帮助模型在分析时优先看到：

- **锚点**：这个系统到底围绕什么在维持或演化
- **约束**：什么在限制它
- **主体层**：谁在维持，谁在推动，谁在卡住
- **推力链**：资源、认同、记忆、协作从哪里来
- **扰动与熵增**：什么在让它失真、僵化、分裂或空心化
- **演化阶段**：它现在处在哪个发展或衰退阶段
- **干预优先级**：先修哪里最值

---

## 这个 skill 做什么

它不是“万物终极解释器”，而是一把结构分析刀。

它最擅长：
- 分析团队为什么卡住
- 分析关系为什么反复失衡
- 分析项目为什么推进困难
- 分析一个制度、社群或组织为什么效率低、内耗高、或开始空心化
- 分析复杂处境中的**核心矛盾、风险方向与干预顺序**

它不擅长：
- 精确预测具体未来事件
- 替代法律、医疗、金融等专业判断
- 在证据很弱时给人贴“本质标签”
- 把模型概念当成攻击工具

## 设计原则

这个 skill 有四条硬原则：

1. **先看结构，再谈对错**  
   优先分析位置、约束、反馈与阶段，不急着裁决人格。

2. **先看高成本信号，再看宣称**  
   重点看持续行为、资源流向、承担与后果，而不是口头表态。

3. **先找约束，再提建议**  
   很多问题不是“没人努力”，而是结构承载已经到了极限。

4. **先给不确定项，再给定论**  
   任何复杂系统都可能存在替代解释。skill 要保留余量。

---

## 你会得到什么

这个 skill 被调用后，最理想的输出不是一篇长篇大论，而是一份结构诊断：

- 当前结构判断
- 核心矛盾
- 主要风险方向
- 最优先干预点
- 不确定项与替代解释

---

## 推荐配合方式

如果你把这个 skill 当成 GitHub 项目长期维护，建议这样迭代：

- `SKILL.md` 只放核心流程与触发条件
- `resources/` 放概念、阶段、护栏
- `templates/` 放输出模版
- `examples/` 放高质量示例
- 每次修订时优先检查：有没有把分析语言写成攻击语言

---

## 一句话定位

**世界结构.skill** 不是世界本身。  
它只是试图用同一套结构语言，去看清复杂对象是怎么形成、维持、失衡、跃迁与崩解的。
