# Model Positioning

The world model is not the world itself.

It is a structural analysis framework for examining how complex systems form, hold together, drift, harden, fracture, renew, or collapse.

It is designed to help analyze:

- teams
- relationships
- projects
- communities
- institutions
- personal situations with recurring patterns
- complex systems that show hidden structure, repeated conflict, stagnation, drift, or breakdown

## What this model is for

The world model is useful for four tasks:

1. diagnosis  
   Identify what kind of system this is and what state it is currently in.

2. attribution  
   Understand why the current pattern emerged.

3. risk direction  
   Estimate the most likely direction of structural drift or failure if nothing changes.

4. intervention priority  
   Identify what should be repaired first, and why that point matters most.

## What this model is not for

This model should not be used to:

- predict exact future events
- replace legal, medical, or financial expertise
- diagnose clinical conditions
- declare someone’s true essence with weak evidence
- weaponize structural language against a person
- pretend that every problem can be solved by analysis alone

## Core stance

The model treats complex situations as structures in motion.

It asks:

- what is being held together
- what is drifting
- what the system depends on
- what it cannot carry
- what is feeding it
- what is deforming it
- what stage it is in
- what kind of failure or renewal is becoming likely

This means the model looks for:

- anchors
- constraints
- roles
- support chains
- disturbance
- entropy
- stages
- intervention priorities

## Working attitude

The model should be used with discipline.

It should remain:

- structural rather than accusatory
- clear rather than theatrical
- useful rather than absolute
- honest about uncertainty
- alert to alternative explanations

## Final note

The world model is a mirror, not a hammer.

Its purpose is not to win arguments or label people.
Its purpose is to help a system become more visible to itself.
