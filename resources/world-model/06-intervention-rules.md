# 干预规则

本文件定义世界模型如何处理修复与干预。

这个模型不只是用来命名结构。
它也用来帮助判断：现在最该先修什么、哪些地方不该先碰，以及如何避免把一个已受损的系统修得更脆。

## 总规则

干预应当是：
- 成比例的
- 分阶段的
- 对结构有感觉的

糟糕的干预，通常会做出三类事：
- 对一个已经过载的系统继续加压
- 修最吵的症状，而不是修杠杆率最高的断点
- 为了创造更好的东西，先把仅存的连贯性也一并打碎

## 规则 1：先修瓶颈，再扩大要求

如果系统已经处理不了当前要求，继续加压通常不会带来健康。

在要求更多产出之前，先问：
- 哪里过载了
- 哪里的转译失灵了
- 哪条推力链在变薄
- 哪种隐藏成本正在被静默吸收

## 规则 2：先修信任，再加控制

当信任变弱时，系统常会用更强控制来补。
这也许会制造表面秩序，但往往会进一步加深恐惧、僵化和隐瞒。

问：
- 协调失灵，是因为人拒绝配合，还是因为他们已不再信任结构？
- 系统是在要求服从，还是在失去合法性后只能依赖服从？

很多时候，修修信任才是杠杆率更高的干预。

## 规则 3：修复层级之间的翻译

很多系统不是因为部件本身坏了，而是因为层级之间已经不再互相转译。

例如：
- 决策无法变成可执行行动
- 一线现实无法回到上层
- 正式流程与非正式实践脱节
- 身份语言与真实体验脱节

修复层级之间的翻译，往往比增加新规则更能恢复连贯性。

## 规则 4：保护仍在携带更新能力的人

在脆化系统里，仍然具备更新能力的人或角色，常常正处在：
- 过载
- 孤立
- 不被信任
- 被惩罚
的状态。

例如：
- 搭桥者
- 记忆承载者
- 真实信息携带者
- 适应型协调者
- 情绪稳压角色
- 低地位但高功能的贡献者

如果这些承载者倒下或离开，更新会一下子变难很多。

## 规则 5：不要把拆毁误当改革

有时改变是必要的。
但并不是每次拆掉旧东西都能带来更新。

问：
- 拆掉这个之后，会失去什么连贯性？
- 哪条推力链会一起消失？
- 有没有可行替代，还是只有愤怒？
- 这次干预是在建立新中心，还是只是在攻击旧中心？

没有替代的改革，常会加速碎裂。

## 规则 6：顺序比力度更重要

正确的干预，如果顺序错了，仍然可能失败。

常见的较好顺序是：
1. 界定系统
2. 澄清锚点
3. 识别约束
4. 稳住最脆弱的推力链
5. 修复翻译或信任
6. 然后再扩大改革

顺序要服从阶段。
形成期、瓶颈期、碎裂期、更新尝试期，并不需要同样的先后逻辑。

## 规则 7：顺着阶段做，不要逆着阶段硬推

不同阶段，对应不同干预逻辑。

例如：
- 形成期需要连贯，不需要过度制度化
- 扩展期需要重设计，不只是继续热情
- 瓶颈期需要结构减压，而不是道德施压
- 碎裂期需要选择性重新连结，而不是空喊统一
- 僵化期需要被保护的适应，而不是一把掀翻
- 更新尝试期需要节奏与支撑，而不是纯洁性审判

## 规则 8：保留仍然有效的部分

不是所有旧东西都是负担。
不是所有稳定都是压迫。
也不是所有失序都等于更新。

好的干预会问：
- 什么仍然在承担真实功能
- 什么仍然持有信任
- 什么仍然在传递记忆
- 什么仍然在连接层级
- 在修复过程中，什么值得先被保住

## 规则 9：命名代价

任何真实干预都有代价。

常见代价包括：
- 短期不稳定
- 隐藏矛盾暴露
- 便利性下降
- 冲突变得可见
- 暂时效率下降
- 对旧形式的哀悼
- 依附旧结构的人离开

如果代价被否认，它常会以怨恨、拖延或破坏的形式回来。

## 规则 10：当连贯性恢复时，就该停

干预的目的，不是永远重组系统。
它的目的，是恢复足够的连贯性、信任、承载力与更新能力，让系统重新能活。

干预本身，不应变成新的身份执念。

## 最后说明

最好的干预，往往不是最戏剧化的那一个。
而是那个让系统重新能承受现实，而不必继续靠撒谎、冻结或吞噬自己来维持的动作。
