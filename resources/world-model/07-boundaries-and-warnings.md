# 边界与警告

本文件定义世界模型的边界、风险与误用提醒。

一个强模型一旦开始被人拿来过度简化现实，它就会变危险。

世界模型只有在边界始终可见时，才真正有用。

## 边界 1：模型不是现实

模型是一副透镜。
它不是被分析的对象本身。

它会突出显示：
- 锚点
- 约束
- 角色
- 推力链
- 扰动
- 漂移
- 熵增
- 阶段
- 干预优先级

但它并不能消除：
- 含混
- 主体能动性
- 偶然性
- 运气
- 深层主观性
- 历史单次性

## 边界 2：模型不能替代专业知识

这个框架不能替代：
- 法律判断
- 医疗知识
- 临床诊断
- 财务分析
- 科学证明
- 具体处境中的直接经验

它能帮助整理思路。
但它不能抹掉领域知识。

## 边界 3：不是所有冲突都结构化

有些问题只是：
- 意外
- 暂时误会
- 个人不相容
- 时机不对
- 孤立失败

不要把每件事都强行塞进宏大结构理论。

## 边界 4：结构不抹除责任

结构诊断可以解释为什么某种行为变得更可能。
但这不自动等于该行为就被免责了。

同样地：
- 一个角色可以被结构扭曲
- 一个人仍然可能在结构中作出选择

不要用“结构”把一切能动性都溶解掉。

## 边界 5：扰动不是污名标签

“扰动”不应被当成：
- 道德侮辱
- 排斥工具
- 压制异议的手段
- 维护权力的借口

它的正确用法是分析性的：
- 什么在持续削弱连贯性？
- 什么在扭曲信任或传导？
- 什么在抬高协调成本，却没有带来更新？

## 边界 6：阶段不是命运

阶段语言是解释性的，不是预言性的。

说一个系统处于瓶颈、僵化或碎裂，并不等于：
- 它已经无法更新
- 它注定毁掉
- 里面的人永远都出不来

阶段分析的用途是增加清晰，不是制造宿命感。

## 边界 7：模型可能被过度使用

一种常见误用，是把模型用得太狠，以至于：
- 细节消失
- 每个人都只剩角色代号
- 每件事都被看成症状
- 分析取代了倾听
- 确定感取代了观察

这不是理论成功，而是纪律失败。

## 边界 8：模型应当暴露不确定性

任何严肃使用，都应给以下东西留位置：
- 盲区
- 缺失证据
- 替代解释
- 局部误读
- 尚未看到的压力

如果一份分析完全不留下不确定性，它往往已经过度僵化。

## 边界 9：模型不应冻结活系统

糟糕的分析，会让系统里的每一个动作都显得像被过度解释，最后使系统更加不敢动。

一个有用的模型，应当：
- 让模式更可见
- 让修复更可能
- 减少混乱
- 收紧优先级

它不应当：
- 让人感觉自己被概念封死
- 把诊断变成支配
- 用无穷解释替代行动

## 边界 10：不要浪漫化痛苦、崩塌与碎裂

痛苦不自动等于深刻。
崩塌不自动等于真相暴露。
碎裂也不自动等于更新的开始。

有些痛苦只是耗损。
有些崩塌只是结构未被及时修复。
有些碎裂只是系统再也承担不了现实。

不要把受损本身美学化。
模型的任务，是帮助识别哪里需要修，而不是赞美裂口。

## 最后警告

当一个模型被用来放大分析者自己，而不是澄清系统时，它就开始变危险。

世界模型应当产出更好的感知，而不是知识虚荣。
